# Switch2GO AAC Contributing Guidelines

Thank you for contributing to Switch2GO AAC!

Contributions from the open-source community are crucial in helping us make the best AAC app possible.

## How to Contribute

### Reporting Bugs
- Check the existing issues to make sure the bug has not already been reported.
- Open an issue describing the bug.
  - Be sure to use a descriptive title and include detailed steps on how to reproduce the issue.
  - Explain the expected behavior vs. the actual behavior.
  - Include the device and OS version.
  - Tag the issue with the `bug` label.

### Suggesting Enhancements
- Open an issue clearly describing the behavior you would like to see.
- If applicable, provide a few examples of how the feature would work and what purpose it would serve.
- Tag the issue with the `enhancement` label.
- **Wait for maintainer approval** before starting implementation.

### Contributing Code

**All code changes require a Pull Request and maintainer review.**

#### Step-by-Step Process

1. **Fork** the repository to your GitHub account
2. **Clone** your fork locally
3. **Create a feature branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** following project coding standards
5. **Test thoroughly** - ensure existing tests pass
6. **Commit** with clear, descriptive messages
7. **Push** to your fork
8. **Open a Pull Request** against the `main` branch

#### Pull Request Requirements

- Clear description of what changed and why
- Link to related issue (if applicable)
- All tests passing
- No secrets, credentials, or sensitive data
- Code follows existing project style

### Review Process

1. A code owner will be automatically assigned to review
2. Reviews typically happen within a few days
3. You may receive feedback or change requests
4. Once approved, a maintainer will merge your changes
5. Your contribution will be included in the next release

## Branch Protection

- Direct pushes to `main` are **not allowed**
- All changes must go through Pull Request review
- Only maintainers can merge approved PRs

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Remember this app serves users with accessibility needs
- No spam, self-promotion, or off-topic contributions

## Questions?

Open an issue with the `question` label.

Thanks for helping make Switch2GO AAC better for everyone!
